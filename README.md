**Epic** (epik) – duży cel lub obszerny temat, który dzieli się na wiele mniejszych zadań / stories.

**Feature** – konkretna funkcjonalność produktu, zwykle obejmująca kilka stories lub zadań.

**Story** (user story) – opis pojedynczej funkcji z perspektywy użytkownika, przynoszącej mu konkretną wartość.

**Task** (zadanie) – jednostka pracy technicznej lub organizacyjnej, niekoniecznie widoczna dla użytkownika.

**Bug** (błąd w programie) – zgłoszenie istniejącej funkcjonalności, która działa nieprawidłowo lub niezgodnie z oczekiwaniami.

**Request** (żądanie / prośba) – zgłoszona potrzeba nowej funkcji, zmiany lub usługi, zwykle pochodząca od użytkownika lub biznesu.
